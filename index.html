The Basics of the Web

How the Web Works

The World Wide Web is a collection of HTML documents. Types of files that can be found on the web are plain text, HTML, Images, Videos and Music.
The major pieces of the web are your computer & browser, The Internet, HTTP, and servers.

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions for example: the computer might do this by making text like this: the results are real.

HTML

HTML (Hypertext Markup Language) is the main type of content and
HTML documents are the heart of the web. It is made up of text content, markup, and references to other documents. Content is what you see; Markup is what the content looks like; and there are references to other documents.

TAGS and ELEMENTS

HTML markup is made up of tags like what is being used to italicize . There is an open tag and a closing tag; For example: contents = bold. An HTML tag is always contain within angled brackets. Most tags have an opening tag (..p..) and a closing tag (../p..).

Some tags (called "void" tags) do not require a closing tag (like the ..br.. tag). An HTML element refers to everything within a set of opening and closing tags. Span and div tags are normal elements used to contain their content. The difference is that Span is inline and div is block. You can attach styles to them to help with the display of your document

An Attribute is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write ..a.. with the ../a.. closing tag href="www.udacity.com"

CSS

CSS controls the "style" of a page. When programmers talk about the DOM they are talking about the tree-like structure of a page. The style is the basically the way that you want your web to look. Like almost everything that exists there are applied rules to every element and repetition is one that should be avoided in all programming; the most applied rule applies to your document. Using HTML classes and CSS style can help you avoid repetition.

Creating a Structure Document with HTML

Developer Tools for HTML elements are either inline or block. Block elements form an invisible box around the content inside of them. The tree-like structure of HTML comes from the fact that HTML elements can have other elements inside of them. The relationship is like a family tree. the structure shows up as a series of nested boxs inside a browser, showing boxes inside of boxes. The relationshiop between indented HTML and Boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it 's corresponding box.
